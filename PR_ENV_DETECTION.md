## Summary

新增三项启动时环境检测功能：**中文路径检测**、**管理员权限检测**与**游戏分辨率检测**，在日志中给出明确提示，帮助用户快速排查因运行环境异常导致的脚本运行问题。同时修复了高 DPI 缩放（150%）下分辨率检测值偏小 1/3 的问题。

## 变更说明

### 1. 中文路径检测

- 启动时检测当前运行目录是否包含中文字符（Unicode 范围 `一-鿿`）。
- 若含中文路径，日志输出黄色 `WARNING`，提示用户移动到纯英文路径下运行，避免部分组件兼容性问题。
- 不阻塞程序执行。

### 2. 管理员权限检测

- 在 Windows 平台启动时调用 `ctypes.windll.shell32.IsUserAnAdmin()` 检测当前进程是否以管理员身份运行。
- 若非管理员权限，日志输出黄色 `WARNING`，提示用户以管理员身份重新运行，但不阻塞程序继续执行。

### 3. 游戏分辨率检测

- 通过进程名 `HTGame.exe` 查找游戏窗口（`find_window_by_process`），再调用 `GetClientRect` 获取客户端区域分辨率（自动排除标题栏与窗口边框）。
- 符合推荐分辨率 `1280x720` 时输出 `INFO` 日志，否则输出 `WARNING` 日志并提示用户调整。
- 未找到游戏窗口或获取尺寸失败时同样给出警告，不影响后续流程。

### 4. DPI 缩放修复

- `win32_process.py` 模块加载时调用 `SetProcessDPIAware()`，使进程声明 DPI 感知。
- **修复前**：系统 150% 缩放时，`GetClientRect` 返回 DPI 虚拟化坐标，实际 1280x720 被误报为 853x480。
- **修复后**：返回物理像素尺寸，分辨率检测结果与实际一致。

## 实现细节

| 功能 | 文件 | 关键 API |
|------|------|----------|
| 中文路径检测 | `agent/main.py:401-411` | `re.search(r"[一-鿿]", cwd)` |
| 权限检测 | `agent/main.py:414-424` | `shell32.IsUserAnAdmin()` |
| 分辨率检测 | `agent/main.py:427-448` | `find_window_by_process()` + `GetClientRect()` |
| DPI 感知 | `agent/utils/win32_process.py:9` | `user32.SetProcessDPIAware()` |
| 窗口查找 | `agent/utils/win32_process.py:48-65` | `EnumWindows` + `GetWindowThreadProcessId` |
| 客户区尺寸 | `agent/utils/win32_process.py:68-72` | `GetClientRect` |
